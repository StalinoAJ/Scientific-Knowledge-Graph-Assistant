Query Lifecycle Sequence

The step-by-step communication between frontend, proxy, backend agents, and the knowledge store.

sequenceDiagram
    participant User as fa:fa-user Researcher
    participant UI as fa:fa-desktop React UI (Cytoscape)
    participant Nginx as fa:fa-server Nginx Proxy
    participant API as fa:fa-bolt FastAPI Backend
    participant DB as fa:fa-project-diagram Neo4j (Graph+Vector)
    participant LLM as fa:fa-microchip Ollama (Llama 3.2)

    User->>UI: Submit Scientific Query
    UI->>Nginx: HTTP POST /api/query
    Nginx->>API: Proxy to FastAPI
    API->>DB: Vector Search (Top-K Similarity)
    DB-->>API: Relevant Document Fragments
    API->>DB: 2-Hop Cypher Traversal (Methods/Authors)
    DB-->>API: Extra Graph Context
    API->>LLM: Augmented Prompt Generation
    LLM-->>API: Synthesized Answer
    API->>Nginx: Return JSON (Answer + Sub-Graph)
    Nginx->>UI: Return Data
    UI->>User: Display Insights & Renders Graph
